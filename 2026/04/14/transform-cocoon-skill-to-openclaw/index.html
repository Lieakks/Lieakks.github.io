OpenClaw 改造 Cocoon AI 架构图 Skill

发表于 2026-04-14 分类于 技术笔记

Cocoon AI 开源的 architecture-diagram-generator 可以把系统描述直接渲染成暗色主题架构图。原版面向 Claude.ai，不能直接在 OpenClaw 中使用。下面记录一次最小可行改造：把它改成 OpenClaw 可调用的 skill，并验证输出效果。

1. 原工具介绍

  • 项目：architecture-diagram-generator
  • 功能：自然语言 → 独立 HTML 架构图（内嵌 SVG）
  • 特点：暗色主题、组件分色、JetBrains Mono 字体、箭头遮挡处理

2. 改造目标

OpenClaw 对 skill 有两条硬要求：

  1. SKILL.md 顶部必须有 YAML frontmatter
  2. 资源文件必须按 skill 目录结构放置，便于运行时读取

因此，这次改造的目标不是重写工具，而是补齐 OpenClaw 兼容层。

3. 改造过程

3.1 补齐 YAML frontmatter

SKILL.md 顶部加入如下声明：

字段说明

字段 作用
name skill 名称，OpenClaw 用它标识调用对象
description 自然语言触发器，决定何时被匹配到
version 版本号，便于维护和回溯
license 许可证信息
metadata.openclaw.emoji 列表中的图标
metadata.openclaw.requires 运行依赖声明；这里为空，表示不依赖额外二进制或环境变量

3.2 调整资源路径

模板文件没有直接写进 SKILL.md，而是放到 OpenClaw 约定的位置：

对应目录结构如下：

assets/ 的作用很直接：放模板、静态资源和后续可能新增的样式文件，避免把大量 HTML 逻辑塞进 SKILL.md。

3.3 保留原设计系统

这次没有改原始视觉方案，只做兼容层改造：

组件类型 颜色
前端 青色
后端 绿色
数据库 紫色
云服务 琥珀色
安全边界 玫瑰色

也就是说，OpenClaw 负责调用，Cocoon AI 负责审美。

4. 结果验证

改造后，OpenClaw 可以正常识别该 skill：

输出中包含：

实际验证时，可以直接把真实项目的目录结构喂给 skill，生成独立 HTML 架构图文件；页面中包含完整 SVG、图例和分层关系，浏览器直接打开即可查看。

5. 结论

这次改造的核心只有三点：

  1. 补 frontmatter，让 OpenClaw 识别 skill
  2. 把模板放到 assets/，让资源路径符合规范
  3. 保留原设计系统，只做兼容层改造

结果是：一个原本面向 Claude.ai 的架构图生成器，变成了 OpenClaw 中可直接复用的技术工具。
